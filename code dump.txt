  <header>
    <div class="title-block">
      <h1>Working with Opus&nbsp;4.8</h1>
      <div class="sub">A one-page operator's guide for building &amp; editing HTML decks that transpose cleanly to PowerPoint (16:9, 1280×720). Treat the LLM like a sharp junior who needs exact instructions — vague in, vague out.</div>
    </div>
    <div class="badge">Internal&nbsp;LLM&nbsp;·&nbsp;Opus&nbsp;4.8</div>
  </header>

  <!-- Golden rule -->
  <div class="golden">
    <strong>The one rule that fixes 80% of bad output:</strong> &nbsp;Don't ask for the whole thing in one go. State the goal, get a draft, then change <em>one thing at a time</em>. Consistency comes from iterating on a working draft — and a consistent deck is what transposes cleanly into PowerPoint.
  </div>

  <!-- Workflow strip -->
  <div class="flow">
    <div class="step"><span class="num">01</span><span class="lbl">Prompt the idea</span><span class="d">Describe what you want + who it's for + the format.</span></div>
    <div class="step"><span class="num">02</span><span class="lbl">Check output</span><span class="d">Open it. Note what's wrong in plain words.</span></div>
    <div class="step"><span class="num">03</span><span class="lbl">Prompt a change</span><span class="d">One fix at a time. Be specific.</span></div>
    <div class="step"><span class="num">04</span><span class="lbl">Iterate</span><span class="d">Repeat 2–3 until you're happy.</span></div>
  </div>

  <div class="cols">

    <!-- Anatomy of a good prompt -->
    <div class="card full">
      <h2><span class="n">1</span>Anatomy of a prompt that works</h2>
      <div class="recipe">
        <span class="lab">ROLE</span>     You are building a slide deck in HTML that will be converted to PowerPoint.<br>
        <span class="lab">GOAL</span>     Create a <span class="var">[5-slide]</span> presentation on <span class="var">[topic]</span> for <span class="var">[audience]</span>.<br>
        <span class="lab">FORMAT</span>   Each slide a fixed <span class="var">1280×720</span> container, absolutely-positioned. One slide per section.<br>
        <span class="lab">STYLE</span>    <span class="var">[corporate / clean / dark]</span>; primary colour <span class="var">[#hex]</span>; named web font <span class="var">[name]</span>.<br>
        <span class="lab">CONTENT</span>  Here is the text for each slide: <span class="var">[paste your bullets]</span>.<br>
        <span class="lab">CONSTRAINTS</span> PowerPoint-safe: real text, solid colours, no gradient-text or backdrop filters.
      </div>
      <ul style="margin-top:7px;columns:2;column-gap:18px;">
        <li>Name the <strong>audience</strong> — it sets the tone automatically.</li>
        <li>Give <strong>exact colours/fonts</strong> as values, not "nice" or "professional".</li>
        <li>Say <strong>"fixed 1280×720, PowerPoint-safe"</strong> so the transpose stays clean.</li>
        <li>Paste real content — don't make it invent your numbers.</li>
      </ul>
    </div>

    <!-- Amending while keeping style -->
    <div class="card amend">
      <h2><span class="n">2</span>Editing a deck <span class="pill">KEEP STYLE</span></h2>
      <ul>
        <li><strong>Paste the existing file</strong> (or attach it) every time — the LLM can't see last week's version.</li>
        <li>Start with: <span class="k">"Keep the 1280×720 slide size, positioning, colours and fonts exactly the same. Only change…"</span></li>
        <li><strong>Never let it change the slide dimensions</strong> or switch to flowing layout — that's what breaks the PowerPoint convert.</li>
        <li>Change <strong>one element per prompt</strong>, then re-check before the next.</li>
        <li>If style drifts, be literal: <span class="k">"You moved the title — put it back to its original position."</span></li>
        <li>Ask it to <strong>return the full file</strong> so you keep one source of truth.</li>
      </ul>
    </div>

    <!-- When output is wrong -->
    <div class="card">
      <h2><span class="n">3</span>When the output is off</h2>
      <ul>
        <li><strong>Describe the symptom, not the fix:</strong> "the text overflows the slide" beats "make the div smaller".</li>
        <li><strong>Paste the error</strong> if something breaks — copy it in verbatim.</li>
        <li><strong>Show, don't tell:</strong> screenshot or describe exactly what you see vs. what you expected.</li>
        <li><strong>Don't restart</strong> — keep iterating in the same chat so it retains context.</li>
        <li>Stuck after 3 tries? Say <span class="k">"start over with a simpler version"</span>.</li>
      </ul>
    </div>

    <!-- Which language to ask for -->
    <div class="card full">
      <h2><span class="n">4</span>Which language to ask for</h2>
      <table>
        <tr><th>Ask for…</th><th>When you want…</th><th>Say something like</th></tr>
        <tr><td>HTML + CSS</td><td>Slides &amp; one-pagers you'll transpose to PowerPoint — the visual layer.</td><td>"HTML deck, 1280×720 slides, PowerPoint-safe."</td></tr>
        <tr><td>JavaScript</td><td>Interactivity — clickable charts, navigation, filters, calculators inside the HTML.</td><td>"Add JS so the chart updates when I pick a year."</td></tr>
        <tr><td>Python</td><td>Reshaping data, automating files, reading/writing Excel so it's clean for the LLM.</td><td>"Python to flatten this Excel into one tidy table per sheet."</td></tr>
        <tr><td>SQL</td><td>Pulling, filtering or aggregating data from a database before charting it.</td><td>"SQL to total revenue by month for 2025."</td></tr>
      </table>
    </div>

    <!-- PowerPoint-safe checklist -->
    <div class="card amend">
      <h2><span class="n">5</span>PowerPoint-safe checklist <span class="pill">TRANSPOSE</span></h2>
      <ul>
        <li>Fixed <strong>1280×720</strong> canvas, every slide identical size.</li>
        <li><strong>Absolute positioning</strong> — not flowing/auto-sizing layout.</li>
        <li><strong>Real text</strong> in elements, never text baked into images.</li>
        <li><strong>Solid hex colours</strong> &amp; named web fonts — no gradient-on-text, no backdrop filters.</li>
        <li>Tables &amp; shapes over CSS tricks with no PPT equivalent.</li>
      </ul>
    </div>

    <!-- Quick wins / phrases -->
    <div class="card">
      <h2><span class="n">6</span>Phrases that raise quality</h2>
      <ul>
        <li><span class="k">"Keep slides 1280×720 and PowerPoint-safe."</span></li>
        <li><span class="k">"Return the complete file."</span></li>
        <li><span class="k">"Match this exact colour palette: …"</span></li>
        <li><span class="k">"Keep everything else unchanged."</span></li>
        <li><span class="k">"Explain what you changed in one line."</span></li>
        <li><strong>Messy Excel?</strong> First: <span class="k">"Python to turn this sheet into a tidy CSV — one header row, no merged cells, no totals."</span></li>
      </ul>
    </div>
