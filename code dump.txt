<!-- Intro -->

<div class="intro">
  <strong>Purpose of this document.</strong> The DAC is set up in Invest One as an SPV with one GTI account representing the sole noteholder (the Investor). This walkthrough translates the S110 DAC mechanics into Invest One terms — instructions, transaction types, and journal entries — so the ops team can process the structure using familiar workflows. The PPN face value plays the role of "units outstanding" and the quarterly coupon plays the role of "distribution." We are using fund language deliberately to bridge the knowledge gap. The underlying mechanics still follow S110 / Trust Deed rules — see the right-column items in the analogy one-pager for what doesn't transfer.
</div>

Invest One will treat it as the capital-like line for capstock reporting, but the underlying GL account is a liability.

Your platform team should confirm which field drives the capstock file output.

  <div class="callout danger">

<h2>Scenario 3 — Cash redemption (principal paydown to Investor)</h2>

<h2>Six key configuration points for the Invest One admin team</h2>

<table class="je">
  <tr><th>#</th><th>Configuration</th><th>Setting</th><th>Why</th></tr>
  <tr><td>1</td><td>SPV / Fund setup</td><td>Single fund, single share class (PPN-A), single GTI account</td><td>One investor, one note, no class proliferation</td></tr>
  <tr><td>2</td><td>Capstock denominator</td><td>Units_Outstanding = PPN face value (not NAV-derived units)</td><td>PPN is debt with a face value, not equity with units</td></tr>
  <tr><td>3</td><td>SUB transaction type</td><td>Increases Units_Outstanding by subscription amount</td><td>Drawdowns increase face value €-for-€</td></tr>
  <tr><td>4</td><td>RED transaction type</td><td>Decreases Units_Outstanding by redemption amount</td><td>Principal paydowns reduce face value €-for-€</td></tr>
  <tr><td>5</td><td>DIV transaction type</td><td>NAV impact: yes. Unit impact: <strong>NO</strong></td><td>Coupon is interest expense — does not reduce face value</td></tr>
  <tr><td>6</td><td>NAV per Unit display</td><td>Will hover around 1.00 throughout life</td><td>Income distributed quarterly — no NAV growth like a typical fund</td></tr>
</table>
