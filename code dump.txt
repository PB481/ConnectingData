# Prompt: Build a Streamlit Section Updater App + Restructured HTML Site

Use this prompt with your internal LLM. Copy the entire block below.

---

## PROMPT START

You are helping me build two things:

1. **A Streamlit application** that allows me to upload a Word document (.docx), select a target section of my existing HTML site, and have the app parse the Word content and inject it into that specific section of the HTML file.

2. **A restructured HTML template** for my site that uses clearly marked, machine-readable section boundaries so the Streamlit app can reliably find and replace content.

### REQUIREMENTS

#### Part 1: The HTML Site Template

Restructure my HTML site so that every updateable section is wrapped in semantic marker comments and a container div. Use this exact pattern:

```html
<!-- SECTION:section_id -->
<div class="updatable-section" id="section_id">
  <!-- Content goes here -->
</div>
<!-- END:section_id -->
```

Create a complete, clean, professional HTML template with the following sections (I will customise the section names later, but use these as defaults):

| Section ID              | Purpose                          |
|-------------------------|----------------------------------|
| `executive_summary`     | High-level overview / headlines  |
| `market_update`         | Market data, commentary          |
| `fund_performance`      | Fund performance tables / stats  |
| `regulatory_update`     | Regulatory changes / alerts      |
| `operational_update`    | Ops metrics, SLA, pipeline       |
| `team_announcements`    | People, org, team news           |
| `key_dates`             | Upcoming dates and deadlines     |

The HTML template must:
- Be a single self-contained HTML file (all CSS inline or in a `<style>` block)
- Use a clean, professional design suitable for internal stakeholders (dark navy header, white content area, light grey background)
- Include a header with a page title and a "Last Updated" timestamp per section
- Be responsive and readable on screen
- Use a sidebar navigation that links to each section via anchor tags
- Include placeholder content in each section so I can see the layout immediately

#### Part 2: The Streamlit App (`section_updater.py`)

Build a Streamlit app with the following functionality:

**Configuration (at the top of the script):**
```python
# Path to the HTML file to update
HTML_FILE_PATH = "site/index.html"

# Section registry — maps display names to section IDs in the HTML
SECTIONS = {
    "Executive Summary": "executive_summary",
    "Market Update": "market_update",
    "Fund Performance": "fund_performance",
    "Regulatory Update": "regulatory_update",
    "Operational Update": "operational_update",
    "Team Announcements": "team_announcements",
    "Key Dates": "key_dates",
}
```

**App Flow:**

1. **Sidebar** with:
   - Dropdown to select which section to update (from `SECTIONS`)
   - File uploader accepting `.docx` files
   - A "Preview" button and an "Update Site" button

2. **Word Document Parsing** (use `mammoth` library as primary, fall back to `python-docx` if needed):
   - Convert the uploaded .docx content to clean HTML
   - Preserve: headings (h1-h4), paragraphs, bold/italic, bullet lists, numbered lists, tables
   - Strip out: images (or optionally extract them to a subfolder), headers/footers, page breaks
   - Clean up any messy markup (empty paragraphs, excessive spans)

3. **Section Replacement Logic:**
   - Read the existing HTML file
   - Use regex to find the content between `<!-- SECTION:section_id -->` and `<!-- END:section_id -->`
   - Replace ONLY the inner content of the `<div class="updatable-section">` — preserve the markers and the div wrapper
   - Automatically update a "Last Updated" timestamp within that section
   - Write the updated HTML file back to disk

4. **Preview Panel** (main area):
   - Show the current content of the selected section (rendered HTML)
   - After uploading a .docx, show a side-by-side or tabbed preview: "Current" vs "New"
   - Use `st.components.v1.html()` to render the HTML preview

5. **Safety Features:**
   - Create a backup of the HTML file before any update (e.g., `index.html.bak` with timestamp)
   - Show a confirmation dialog before writing
   - Display a success/error message after update
   - Log all updates with timestamp, section, and filename to a simple `update_log.csv`

**Key Python Code Patterns to Use:**

For section replacement:
```python
import re
from datetime import datetime

def replace_section(html_content: str, section_id: str, new_inner_html: str) -> str:
    """Replace the inner content of a section while preserving markers."""
    timestamp = datetime.now().strftime("%Y-%m-%d %H:%M")
    
    # Add timestamp to the new content
    timestamped_content = f'<div class="updatable-section" id="{section_id}">\n'
    timestamped_content += f'  <p class="last-updated">Last updated: {timestamp}</p>\n'
    timestamped_content += f'  {new_inner_html}\n'
    timestamped_content += f'</div>'
    
    pattern = rf'(<!-- SECTION:{section_id} -->)\s*<div class="updatable-section" id="{section_id}">.*?</div>\s*(<!-- END:{section_id} -->)'
    
    replacement = rf'\1\n{timestamped_content}\n\2'
    
    updated = re.sub(pattern, replacement, html_content, flags=re.DOTALL)
    return updated
```

For Word-to-HTML conversion:
```python
import mammoth

def convert_docx_to_html(uploaded_file) -> str:
    """Convert uploaded .docx to clean HTML."""
    result = mammoth.convert_to_html(uploaded_file)
    html = result.value
    messages = result.messages  # warnings
    return html, messages
```

#### Part 3: Word Document Authoring Guide

Also produce a short markdown guide (`AUTHORING_GUIDE.md`) that explains to content authors how to structure their Word documents for best results. Include:

- Use Heading 1 for section titles (these will become `<h1>` tags)
- Use Heading 2/3 for sub-sections
- Use bullet lists and numbered lists normally
- Use simple tables (no merged cells — they convert poorly)
- Avoid text boxes, SmartArt, and embedded objects
- Avoid manual formatting (e.g., manually bolding text to look like a heading instead of using a heading style)
- Keep images simple or omit them (the app will strip them by default)
- One document per section update — don't put multiple sections in one doc
